# 03. マダミスシナリオの作成方法論

**調査日: 2026-06-15**
**対象**: 日本語圏の同人・商業マダミス制作者によるブログ・note・ガイド記事

ゼロから作る手順とコツのノート。構成要素そのものの分解は `02_components.md` を参照。

---

## 0. 概観

制作に「唯一の正解」はなく、出発点も「テーマ／キャラ／舞台設定」のどれからでも成立する。ただし共通して語られる**制作フロー・設計原則・落とし穴**が存在する。

---

## 1. 全体制作フロー

### 1-1. コンセプト・テーマ決め（起点）

- **テーマ** = 「プレイし終わったときにプレイヤーの心に残る感情」
- **コンセプト** = 「もし◯◯をマダミスで作るなら？」という問い
- キャッチコピーを「◯◯×◯◯×◯◯マダミス」形式にすると軸がブレにくい
- 出発点はどれでもよい（テーマ中心／キャラ中心／舞台中心）。専門家でも「キャラから派」と「コア面白さ→環境→キャラの逆順派」に見解が割れている
- 初作は **「1時間程度・4人前後」** から始めると現実的（テストプレイヤーを集めやすい）

### 1-2. 事件・真相の設計（結末から逆算）

- 「始めから順に作ると選択肢が増えすぎて脱線する」ため**結末から逆算**する
- 5W1H（Who/What/When/Where/Why/How）を全て埋める
- フーダニット（Who）だけでなく「アリバイ（When/Where）・手法（How）・動機（Why）」を多層化するほど推理に奥行きが出る
- 「実は別の犯人がいた」系の二段構えが事件を魅力的にする

### 1-3. キャラクター・関係性の設計

- 全キャラに「強い動機」「少なくとも1人との強い関係性」「事件への介入度」を付与
- 類似キャラを避け個性を際立たせる（ステレオタイプ的な分かりやすさは有効）
- 犯人以外の全員にも秘密を持たせる（事件に関連し、公開時に議論を変える内容が理想）

### 1-4. タイムテーブル・情報配置 ★最重要

- **タイムテーブルはExcelで管理**し、各キャラの時系列行動を記録（「13:00 ランチから帰宅」等）
- タイムスケジュール表を作らずに執筆すると登場人物間の矛盾が頻発する
- 情報は「全員共有 / 個別秘匿 / フェーズ解禁 / 条件付きカード」の4種に分類して設計

### 1-5. ハンドアウト執筆

- 基本進行：オープニング → 議論① → 追加情報 → 議論② → エンディング
- **一人称**で書く（二人称は設定過多になりやすい）
- **犯人のHO情報量は他キャラの1.5〜2倍**推奨（話せる秘密がなくなる問題を防ぐ）
- 各キャラに個別エンディング描写を用意（「自分のエンディングを聞いて初めて物語が完結した」体験）
- HOを開いた瞬間の「ワクワク感」を重視する

### 1-6. テストプレイ・調整

最低3回。各回の指摘を言語化・記録し、修正方針を決めてから改定する（詳細は §6）。

## 2. 事件・トリック・アリバイの設計

### 2-1. 真相の構築

| 要素 | 設計指針 |
|------|----------|
| 犯行凶器 | 証拠情報として整理し、犯人のみが知る事実と結びつける |
| タイムテーブル | 全キャラの時系列行動を矛盾なく確定させてから執筆 |
| 動機 | 「登場人物にとって強い思い入れのある対象」に関連させると没入度が上がる |
| 犯人確認情報 | 「犯人のみぞ知る情報の自白」が最終的な確認手段として必須 |

### 2-2. 推理が成立するための手がかり配置

- 凶器・タイムテーブル・動機の3要素のうち**2つまでしか公開されない**設計で緊張感を維持
- 最終投票前に「最低でも犯人候補2人程度に絞れる情報」を全員が持てるよう配置（絞れない状態は運ゲー化）
- 時刻は「夜に起きた」でなく「22:15」のように**正確に明記**する

### 2-3. ロジック破綻の防止

- 医学的・論理的・心理学的な粗は致命的になりやすく事前に吟味
- ただし完全リアルも危険。フィクションとしての説得力を保ちつつ適度に抽象化
- 「ゲーム中に気づくことが不可能な後出し要素」「突飛で稚拙なトリック」は低評価に直結

---

## 3. 情報設計

### 3-1. 原則 — 情報の非対称性

マダミスの面白さの本質は「情報の非対称性」。「知っているからこそ有利／知らないと不利になるルール」が面白さを作る。

### 3-2. 情報の種類と配置

| 情報種別 | 特徴 |
|---------|------|
| オープニング情報 | 全員共有、事件発生の前提 |
| キャラクターシート | 個別秘匿、初期情報。控えめにして後発情報の衝撃を高める戦略が有効 |
| 追加シート（パラダイムシフト用） | 後半に開示、議論の構造を覆す |
| カード類 | ランダム性・条件付き、場の動的変化を生む |
| 特殊ルール | フェーズ依存、「新しい死体」など展開刷新 |

「キャラクターシートの情報と後入手情報のバランスをどうとるかがゲームデザインの大きな肝」。

### 3-3. ミスリードの作り方

- 犯人の個別ミッションに「事件と関連の薄い要素」を織り交ぜて推理を錯乱させる
- 表面上は共有に見える情報を実は個別情報にする
- ただし「気づけない後出し」「伏線ゼロの展開」はミスリードでなく欠陥

### 3-4. 手がかりの分散配置

- プレイヤーの解き方は「情報整理→時系列整理→動機整理」の順
- これに対応するよう動機・時系列を複雑にし、情報をばらけさせる

---

## 4. 「全員に役割と物語を持たせる」設計

### 4-1. 基本原則

- 「（前もってアナウンスしない限り）1人を犠牲にすることはあってはならない」（中村誠）
- どのキャラも主人公級の魅力を持つべき。「数合わせのバッファー」と感じさせない

### 4-2. 無実プレイヤーへの目標設定

| 設計要素 | 内容 |
|---------|------|
| 当事者性 | 被害者の家族・関係者など、事件への強い関与を持たせる |
| 調査権限 | 警察・探偵などの公的立場で動機を明確化 |
| 個人ミッション | 「犯人を見つける」以外の秘密目的を付与してジレンマを演出 |
| 強い関係性 | 各キャラが最低1人とは強い関係を持ち孤立を防止 |

### 4-3. ミッション設計の効果

- 対立ミッションでプレイヤー間の葛藤が生まれ、情報共有か秘匿かの駆け引きが発生
- ミッションが「明確な目的を与え迷走を防止」する
- 犯人判定の確率は約30%程度が目安（全員が均等に疑われる設計）

### 4-4. 指向性の明示

- 「ロールプレイ重視」か「推理重視」かを明確化し事前開示することで「Not for me問題」を回避
- 推理重視ならトリックの面白さ、RP重視なら人物描写の深さに注力

---

## 5. よくある失敗・落とし穴

### 5-1. 情報設計の失敗

| パターン | 具体内容 |
|---------|---------|
| 情報過多 | 多すぎて整理できず議論が拡散 |
| 情報過少 | 全情報を合わせても絞れない → 投票が運ゲー化 |
| 時刻の曖昧さ | 「夜に起きた」等。正確な時刻が必須 |
| 後出し真相 | ゲーム中に気づけない情報を根拠にしたトリック |
| 情報量の不公平 | 犯人HOの情報量が少なく「話せる秘密がない」 |
| 登場人物間の矛盾 | 時系列未設計のまま執筆すると相互矛盾が多発 |

### 5-2. 構成・設計の失敗

| パターン | 具体内容 |
|---------|---------|
| 推理が一本道 | 手がかりが唯一の正解へ直結し謎解きの醍醐味がない |
| 推理が不可能 | ヒント不足・隠蔽されすぎて議論が進まない |
| バッドエンドの手薄さ | グッドエンドのみ充実。分岐全体の満足感設計が必要 |
| 手持ち無沙汰なキャラ | 目標・関係性・秘密が薄いキャラができ1人が退屈 |
| 世界観のニッチ化 | 専門知識前提で共感できない |
| 難易度過剰 | 初稿は「激ムズ」になりがち。テスト後に大幅下げが必要なことが多い |
| 導入の納得感欠如 | 「警察に通報すればいい」という疑問を払拭する設定不足 |

### 5-3. ロジック面の失敗

- 医学的・法的・物理的に無理のある犯行手法（フィクションの説得力の範囲で適度に抽象化）
- 「完璧な論理」より「プレイ中の夢中状態の維持」が優先。小さな違和感は事後に許容されやすい

## 6. テストプレイの重要性と進め方

### 6-1. 位置づけ

- 制作の3つの山は「①シナリオ制作 ②テストプレイ ③リリース」で、テストプレイが「一番大変」
- 1回限りのゲーム体験のため満足度に取り返しがつかない。「作品の防御力を高める最重要工程」

### 6-2. 実施方法

- **最低3回**実施が推奨
- 各回後に「指摘を言語化して記録」し、修正方針を確定してから改定
- 感想はクールダウン時間を設けてから記録すると建設的なフィードバックが得られる

### 6-3. 心構え

- 「ほぼ確実に批判的意見が飛んでくる」前提で臨む
- 完成度を「複数のバケツ」として段階的に考えると心理的負荷が軽減
- 作者の自信のなさは作品の魅力を損なうため姿勢を整える

### 6-4. デザイン面の充実

- ユドナリウムやイラストなど視覚要素は没入感とフィードバックの質を高める
- 血液描写・ゲーム的要素など事前に知りたい情報を詳細に開示（「Not For Me判定」の回避）

## 7. 工数 / 商業と同人の違い / 著作権

### 7-1. 制作工数の目安

- 初心者が4〜5人用シナリオを初めて作る場合、数ヶ月規模になることが多い
- 流れ：制作（数週間〜数ヶ月）→ テストプレイ（複数回）→ 修正（各1ヶ月程度）→ リリース
- 修正フェーズが想定外に長引くケースが多い（難易度の大幅下げ等）

### 7-2. 商業 vs 同人

| 項目 | 商業 | 同人 |
|-----|------|------|
| 品質担保 | プラットフォーム審査あり | 自己責任 |
| 流通 | 専用プラットフォーム（ウズスタジオ等）、カフェ | BOOTH・フリー配布・SNS |
| 収益 | 手数料控除後に還元 | 直接販売 or 無料 |
| 宣伝 | プラットフォームの集客力 | SNSでの自力宣伝が基本 |
| 縛り | 規約・審査基準 | 比較的自由だが権利処理は自己管理 |

### 7-3. 著作権・権利の注意（グレーゾーンが多く発展途上）

**何が著作物か**
- 著作物：ストーリー・パッケージデザイン・挿絵
- アイデア（保護外）：世界観・キャラ設定・ゲームルール

**同人（非営利個人利用）**
- オフライン：5〜10名程度の非営利上演は権利者許可なしで可能（演奏権等の例外）。ただしシナリオの複製コピーは禁止される可能性が高い
- オンライン：公衆送信権・複製権の許諾が必要な可能性。権利者が明示していない場合は本来NG

**商用利用**
- 「商用利用可」表記だけでは不十分。オフライン/オンライン区分・複製可否・改変許容度・配信可否を個別確認
- 現場での改変は「翻案権」「同一性保持権」侵害になりうる

**推奨**
- 自作公開時は利用規約に「個人利用版/商用利用版」の区別と具体的OK/NGを明記
- 既存IP（漫画・映画等）を題材にする場合は二次創作ガイドラインを確認
