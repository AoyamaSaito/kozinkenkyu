# 09b. P4 証拠カード ── 第2調査フェーズ（6枚＋固有マス2枚）

> **位置づけ**: P4（第2調査フェーズ）で探索マスから取得する通常証拠カード6枚の本文。
> 固有マス札（金庫→論文①、認証扉→論文②）は別管理（`07_evidence_catalog.md`）。
>
> **カード仕様**:
> - 通常カード: `make_evidence_card_wa(variant="ofuda")` / 600×840px / font23 / 18文字折返し / 行高36
> - 灰の残骸: `make_shock_card` / 600×840px / font25 / 16文字折返し / 行高40 / **イラスト領域ありのため本文4行まで**
>
> **P4全体のゴール**: 「入れ替わり（なりかわり）が起きている」を確立する＋施設深部の異常を叩き込む
>
> **関連**: `07_evidence_catalog.md`（証拠品カタログ）/ `05_clues.md`（消去法格子）/ `09a_evidence_p1.md`（P1カード）

## P4-01. 破損した拘束具

| 項目 | 値 |
|------|-----|
| header | 物的証拠 |
| prefix | C |
| No. | 09 |
| title | 破損した拘束具 |
| 分類 | 雰囲気 |

### カード本文

```
区画の奥に据えられた金属の椅子。
両腕と両足に拘束ベルト。
片方のベルトが
根元から引きちぎられている。
座面に経年の汚れ。
長い間使われていた形跡がある。
```

### 設計意図

- 「何かを拘束して研究していた」ことの視覚的証拠。P1-05（研究日誌）の「接触時の反応パターンを記録」が何を意味していたか、ここで具体化される
- 「引きちぎられた」＝拘束を破って脱出した暴力性。Bの脱出を間接的に示唆
- P1（「化け物がいた」）→ P4（「化け物を閉じ込めて研究していた、それが出た」）への進行

## P4-02. 先遣隊の最終手記

| 項目 | 値 |
|------|-----|
| header | 記録 |
| prefix | L |
| No. | 10 |
| title | 先遣隊の最終手記 |
| 分類 | 情報（遭遇→途絶） |

### カード本文

```
[手書き・走り書き]
奥の区画で遭遇。
接触を試みたが
反応が予測と異なる。
退避を──

ここでインクが途切れている。
ノートの残りは白紙。
```

### 設計意図

- P1-02（先遣隊の活動記録）の続き。P1は淡々とした業務報告だったが、P4では切迫した走り書き
- 「接触を試みた」＝先遣隊は何かに遭遇して接触を試みた。「反応が予測と異なる」＝想定外の事態→覗きによる崩壊の前触れ
- 途中で止まるインク＝先遣隊が突然消えた（崩壊した）ことの生々しい表現
- P1-02（第3日の報告→途絶）から P4-02（最後の瞬間の記録）へ、段階的に詳細度が上がる

> **GM注**: 「遭遇」した相手はB（封印から出た直後の上位存在）。先遣隊はBに接触→覗かれて崩壊した。

## P4-03. 灰の残骸

| 項目 | 値 |
|------|-----|
| header | 物的証拠 |
| prefix | C |
| No. | 11 |
| title | 灰の残骸 |
| 分類 | **衝撃**（専用フォーマット） |
| フォーマット | `make_shock_card` — 暗転色調。**本文4行制限** |

### カード本文

```
隔離区画の奥に、灰の塊。
先遣隊と違い、
人の原型すらとどめていない。
ただの灰。
```

### 設計意図

- **P4の中心証拠**。「入れ替わりが起きている」を物理的に示す最も重要なカード
- 崩壊度の差＝「この1体だけ他と違う崩れ方をしている」。他の先遣隊員はフォルム残存だが、これだけ灰のみ＝**器として完全に使い切られた**（＝Bの前の器）
- 論文①（金庫）が「崩壊度の段階記録」を提供 → このカードの意味が深まる（「なぜ1体だけ違うか」の理論的説明）
- 事実だけ並べて「はず。」で止める。結論はPLの頭の中で生まれる設計。査読フックはカードに入れず、論文①との接続は自然発生に任せる（必要ならGMの口頭補足で処理）
- **カード間接続**: P1-03（防護服のロッカー/「焦げた跡」）→ P4-03（灰の残骸/崩壊の物証）。P1で見た「焦げ跡」の意味がここで分かる

## P4-04. 封鎖プロトコル通知

| 項目 | 値 |
|------|-----|
| header | 記録 |
| prefix | L |
| No. | 12 |
| title | 封鎖プロトコル通知 |
| 分類 | 情報（封鎖の原因＋時系列） |

### カード本文

```
[施設管理システム]

20██/5/12 通信喪失を検知。
プロトコルF-7を発動。
外周封鎖──完了。

解除条件: ██████████
```

### 設計意図

- PLが「なぜ自分たちは出られないのか」を理解するカード。Phase 0で「出口封鎖」は体験しているが、**原因**はここで初めて判明
- 時系列の手がかり: 通信喪失＝先遣隊が全滅した時点。日付がP1-02（活動記録）/ P1-06（収容監視ログ）と一致 → 「あの日に何かが起きた→通信途絶→施設封鎖」の再構成
- **「解除条件」が墨消し**＝PLの好奇心を煽る。答えは論文④（P7・封印装置で封印すれば封鎖解除）
- Bの新タイムラインとの整合: この自動封鎖がBを施設内に閉じ込めた原因

> **GM注**: 日付はP1-02、P1-06、P4-06（走査ログ）と一致させること。

## P4-05. 散乱した調査装備

| 項目 | 値 |
|------|-----|
| header | 物的証拠 |
| prefix | C |
| No. | 13 |
| title | 散乱した調査装備 |
| 分類 | 雰囲気 |

### カード本文

```
先遣隊のものと思われる装備。
測定機器、記録媒体、通信端末。
床に散乱している。
一部はまだ電源が入ったまま。
持ち主が突然
いなくなったかのよう。
```

### 設計意図

- 先遣隊が「突然消えた（崩壊した）」ことの間接的な物証。灰の残骸が「1体の崩壊」なら、こちらは「場全体の異常」
- 「まだ電源が入ったまま」＝それほど最近の出来事。先遣隊が来たのはそう遠い過去ではない
- 「回収した形跡がない」＝先遣隊の後に誰もここに来ていない（…はずだが、Bは来ている。この矛盾に気づくPLがいれば鋭い）
- 雰囲気系カードだが、「先遣隊が突然全滅した」事実を補強する情報的機能も持つ

## P4-06. 走査装置の動作ログ

| 項目 | 値 |
|------|-----|
| header | 記録 |
| prefix | L |
| No. | 14 |
| title | 走査装置の動作ログ |
| 分類 | 情報（遅延回収型） |

### カード本文

```
[自動走査ログ]
設置場所: 区画間認証機構

20██/5/10 走査実行──非該当
20██/5/11 走査実行──非該当
20██/5/12 走査実行──該当(1)
──区画間移動を検出──
```

### 設計意図

- **P7で回収される伏線カード**。P4時点では「何かが検知されて動いた」としか分からないが、P7で論文④（認証扉＝空洞検知装置）が判明すると：
  - 「非該当」＝人間（求心力あり）が通過 → 反応なし
  - 「該当(1)」＝**空洞体（上位存在）が1体、区画間を移動した**
  - 日付が先遣隊事件日と一致 → **Bが封印から脱出して施設内を移動した記録**
- 新Bタイムライン（封鎖中に施設を探索した）の物的裏付け
- 「認証機構」の文言が自動認証扉（固有マス札）とリンク → PLが扉カードと突き合わせる動機
- P1-06（収容監視ログ「Loss detected」）と対：あちらは「出た」瞬間、こちらは「動いた」記録

> **GM注**: 「該当」の日付はP1-02/P1-06/P4-04と一致させること。「非該当」の日付は施設運用中（先遣隊以前）の日付。

## カード間の接続

```
P4-01 拘束具 ──────────→ 「拘束して研究していた＝収容の現場」

　　　　　　　　　　　　　　└→ P4-03 灰（拘束を破って出た存在の末路）

P4-02 最終手記 ──────→ 「先遣隊は何かに遭遇して消えた」
　　　│　　　　　　　　　├→ P1-02 活動記録（業務報告→切迫した走り書き）
　　　│
P4-03 灰の残骸（★中心）→ 「1体だけ違う崩れ方＝入れ替わりの物証」
　　　│　　　　　　　　　├→ 金庫・論文①（崩壊度の段階記録＝理論的説明）
　　　│　　　　　　　　　└→ P1-03 防護服ロッカー（「焦げ跡」の回収）
　　　│
P4-04 封鎖通知 ──────→ 「通信途絶で施設が自動封鎖」
　　　│　　　　　　　　　├→ P1-06 収容ログ（日付一致＝「出た→途絶→封鎖」）
　　　│　　　　　　　　　└→ 論文④（P7）（「解除条件」の答え＝封印完了）
　　　│
P4-05 散乱装備 ──────→ 「先遣隊は突然全滅した」（雰囲気補強）
　　　│
P4-06 走査ログ ──────→ 【P7で回収】「空洞体が移動した記録＝B脱出の裏付け」
　　　　　　　　　　　　　├→ 認証扉（固有マス）との対応
　　　　　　　　　　　　　└→ 論文④（P7）（「認証＝空洞検知」で意味が判明）
```
