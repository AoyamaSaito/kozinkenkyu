# 02. マダミスシナリオの構成要素

**調査日: 2026-06-15**

シナリオが「何の部品でできているか」を分解したノート。作成方法論は `03_scenario_design.md` を参照。

---

## 1. ハンドアウト（HO）の構造

ハンドアウト（キャラクターシート・設定書とも）は、各プレイヤーに個別配布される**秘匿資料**。そのプレイヤーだけが読む前提で設計され、他PLに見せるのはマナー違反。

### 標準的な記載項目

| 項目 | 内容 |
|------|------|
| キャラクター設定 | どんな人物か（職業・年齢・性格・バックストーリー） |
| 秘密 | 事件に関連する隠したい事実（例「被害者を殴った」「宝石を盗んだ」）。原則非公開 |
| 目的／ミッション | メインミッション（犯人当て or バレ防止）＋個人サブミッション |
| 事件前後のタイムライン | 事件当日に自キャラが何をしていたか（アリバイ・行動記録） |
| 人間関係 | 他キャラへの印象・既知の関係 |
| 知っていること | ゲーム開始時点で把握している事実 |
| 勝利条件 | ミッション達成の判定基準 |

### 秘密の「公開レベル」設計（TRPGノウハウからの転用）

- レベル1: 積極的に公開 / レベル3: できるだけ隠す / レベル5: 絶対に隠す
- 「公開時の影響」「公開の条件・タイミング」を明記すると、プレイヤーの迷いが減る

---

## 2. 共通情報と個別情報の分け方

### 共通情報（全員に配布される前提知識）

| 要素 | 説明 |
|------|------|
| 舞台設定 | 時代・場所・空間の説明 |
| 世界観・用語集 | ゲーム固有の設定や特殊ルール |
| 事件の状況（目に見える事実） | 被害者・事件発生状況など表面的な情報 |
| 登場人物表 | 参加キャラ一覧（名前・関係など外面情報） |
| 目に見える手がかり | 現場の物的証拠の概要 |
| ルール説明 | 議論形式・嘘の可否・カード取得ルール・投票方法 |

### 個別情報（各プレイヤーのみが知る情報）

- HOに記載された秘密・タイムライン・目的
- 議論中は口頭共有しか認められず、情報を出すかはPLの判断に委ねられる
- **全員の個別情報を合算すると真相が解明できる構造**が基本（＝真相の分散設計）

### 設計のポイント — 情報表

- 誰が何を知っているかを管理する「**情報表**」（縦軸=情報項目 × 横軸=キャラクター）でバランスを可視化する
- 1セルあたり1つの情報を割り当てて調整する

## 3. フェーズ構成の設計

### 標準的なフェーズ構成

```
[オープニング]   世界観・ルール説明・配役・HO読み込み（15分程度）
[調査フェーズ]   情報カード・アイテム取得・現場調査（カードは1人3〜6枚が目安）
[議論フェーズ]   複数回構成が多い。全体議論 / 密談 / 全体＋密談可 / 自由（1ラウンド10分前後が一例）
[追加情報提供]   中盤以降に情報カード山札を変化／追加HOを挿入して劇的転換
[アクションフェーズ]（任意）特殊能力の使用・エンディング変化に直結する自由行動
[投票フェーズ]   犯人と思う人物に投票。同票処理・推理披露順も事前設計
[エンディング]   投票結果発表 → 真相開示 → キャラ別エンディング → GMネタバレ解説（約20分）
```

### 議論形式の種類（設計者が選ぶ）

| 形式 | 内容 |
|------|------|
| 全体議論 | 全員が同じ場で話す |
| 密談 | 2〜数名の秘密会話（参加人数制限を設計） |
| 全体＋密談可 | 全体進行中に密談も許容 |
| 自由 | 制限なし自由行動 |

### 情報カードの設計

- 引くタイミングは「議論中に自由に引ける」ルールが推奨（順番制は集中力を損なう）
- 全カードを全員が公開すると犯人が特定できてしまう場合、全体公開や見せ合いに制限が必要
- フェーズごとの引き枚数・連続引き制限・公開時の所有権変化・譲渡ルールも設計要素

## 4. キャラクター設計 — 「全員が主役」の原則

### 核心

- 犯人を含む全キャラに「そのキャラならではの物語・秘密・目的」を持たせる
- 孤立キャラ（関係性が薄い／情報を持ちすぎる・持たなすぎる）は設計の失敗

### キャラクター設計の要素

| 要素 | 内容 |
|------|------|
| 立ち位置 | 事件への当事者性（被害者との関係・捜査目的・犯人側/解決側の勢力） |
| 行動原理 | PLが理解・共感できる動機。行動の根本にある感情 |
| 秘密 | 意外性があり事件に関連するもの。複数キャラ間で共有される秘密はより効果的 |
| サブミッション | キャラ固有の個人目標。メインミッションと並行 |
| 人間関係の網 | 相関図・マトリクスで全キャラ間の関係を設計。孤立キャラを作らない |

### 犯人キャラの設計

- 犯行の動機・方法・証拠の設計
- 犯人ミッション＝「バレないこと」がメイン
- 他キャラが手がかりを集めると犯人を絞れる構造にする
- 難度調整: 犯人特定確率30%程度に設定する制作者もいる

### ミッションのジレンマ設計

- 対立ミッションを複数キャラ間に配置し、意図的な衝突・葛藤を発生させる
- 全配役で難度がほぼ均等になるよう調整（ハズレキャラを出さない）

---

## 5. 勝利条件の設計

### 多層構造

「犯人を当てる/バレない」の一軸ではなく、複数の個人目標が同時に走る多層構造が基本。

```
メインミッション（全員共通）
  ├─ 犯人以外: 犯人に最多票を入れる
  └─ 犯人:     自分への投票を避ける
サブミッション（個人目標・秘密）
  ├─ 「Aと和解する」
  ├─ 「Bの持つ書類を奪う」
  └─ 「自分の別の秘密をバレずに終える」など
秘密の保持（全員）
  └─ 各自の秘密を暴かれずにゲームを終える
```

### エンディング分岐

- 投票結果（正犯人が当たったか）で分岐
- アクションフェーズの選択もエンディングに直結
- 各キャラのミッション達成状況で個別エンディングが変化
- 「自分の選択が結果を変えた」体験を提供する

### 注意

- 勝利条件が曖昧だと「秘密を守れたと思い込んで終える」ケースが多発 → 達成条件を**具体的・明文化**する
- サブミッションは「得点向上」と「議論の迷走防止」の二機能を持つ

## 6. GM用の進行台本・タイムテーブルに含まれる要素

### 台本に含まれる情報

| 項目 | 内容 |
|------|------|
| 全HO内容の把握 | 全PLの秘密・ミッション・タイムラインを熟知 |
| **タイムテーブル**（事件内時系列） | 縦軸=時刻/イベント × 横軸=キャラ・被害者・重要アイテム・NPC。誰がどこで何をしたかを俯瞰 |
| **情報表**（誰が何を知っているか） | 縦軸=情報項目 × 横軸=キャラ |
| フェーズ進行スケジュール | 各フェーズの時間配分（読み込み15分／議論×2／追加情報3分／投票5分／EN10分 等） |
| カード・追加HOの配布タイミング | どのフェーズで何を追加配布するか |
| 朗読文・導入セリフ | オープニング・各フェーズ移行時のGM読み上げテキスト |
| エンディング分岐対応表 | 投票結果・ミッション達成状況別のパターン |
| ネタバレ解説資料 | 約20分分の真相解説（動機・トリック・各キャラの裏事情） |
| 想定外への対応メモ | PLが想定外の推理をした場合の対応方針 |

### GMの役割

司会進行・ルール説明・時間管理・初心者の疑問対応・問題行動（メタ推理等）への対処・エンディング/ネタバレ解説。

### GMレス（GM不要）シナリオの場合

- HOや導入文書をより明確・視覚的に設計する必要がある
- 各フェーズの移行条件をシナリオ内に明文化し、進行が自己完結するようにする
