# udonarium_test — 成果物カタログ & デザイン仕様

> マダミス『覗きの代』の素材検証フォルダ。
> ①ユドナリウム取り込み用データ（駒・盤面・カード）の生成検証と、②配布物のデザイン試作を行う。
> **すべての成果物は2本のスクリプトから再生成可能**（このファイルが人間可読の仕様書、スクリプトが実行可能な基盤）。
> 最終更新: 2026-06-17

## フォルダ構成

```
udonarium_test/
├── build_udon.py        # ユドナリウム取り込み用zipを生成（駒・盤面・カード）
```

## スクリプトと再生成手順

| スクリプト | 役割 | 実行 | 出力先 |
|-----------|------|------|--------|
| `build_udon.py` | ユドナリウムのオブジェクトzipを生成＋自己機械検証(C1〜C4) | `python build_udon.py` | `outputs/udonarium/` |
| `design_card.py` | 証拠カード(表/裏)・研究論文（**確定: mincho/warm/2col**＋探索用mono）を生成 | `python design_card.py` | `outputs/design/` |
| `design_paper_styles.py` | 研究論文の**配色7種**を探索→コンタクト | `python design_paper_styles.py` | `outputs/design/` |
| `design_layouts.py` | 研究論文の**レイアウト4種・フォント5種**を探索→コンタクト | `python design_layouts.py` | `outputs/design/` |

- 依存: Python 3.12 / Pillow（RAQM有効）。**cairosvg等は不要**（PILのみ）。
- フォントは Windows 標準（下記「デザイン仕様」参照）。

## outputs/udonarium/ — ユドナリウム取り込み用zip

| ファイル | オブジェクト | 内容 | 実機表示 |
|---------|------------|------|----------|
| `map_table.zip` | game-table | 地下研究施設マップ（エントランス/研究室/資料保管庫/観察室/隔離区画/封印室） | 前回確認済 |
| `char_pc1.zip` | character | PC1 民俗学者（巻物アイコン） | 〃 |
| `char_pc2.zip` | character | PC2 調査隊員（懐中電灯） | 〃 |
| `char_pc3.zip` | character | PC3 大学教授（角帽） | 〃 |
| `char_pc4.zip` | character | PC4 大学院生（本） | 〃 |
| `clue_card.zip` | card | 裏表カードのサンプル（front/back両面とも自前画像） | 〃 |

- 駒は **正体（善A/悪B）を出さず職種のみ**。ステータス・チャットパレットは無し（detailは空ノード）。
- 取り込み: ユドナリウムで各zipをドラッグ＆ドロップ。配置調整・room保存はGM側がGUIで行う運用。

### ユドナリウム正解XML構造（実機エクスポートとの照合で確定）
- **game-table**: 属性方式・子要素なし。`imageIdentifier` は**拡張子なしハッシュ**。
- **character**（単品）: ラッパー無し。ルート直下に `<data name="image/common/detail">`。`imageIdentifier` は拡張子なしハッシュ（要素値）。
- **card**（単品）: `<node>`方式。`front`/`back` の値は **`ハッシュ.png`（拡張子付き）**。
- **room一括保存**は別構造（`<room>`下に `<data name="card">` ラッパー方式）。GUI保存時はツールが自動生成するため当スクリプトでは扱わない。
- zip共通: フラット構造 / `data.xml`(UTF-8・XML宣言) / 画像名 = SHA-256hex + 拡張子。
- 機械検証 C1=フラット / C2=well-formed / C3=画像名一致 / C4=参照⊆画像。

### デザイン言語（確定）「科学 × 日本神話/民俗学」
- **テーマ**: 研究所が伝承（魂抜き・依代・封印）を**標本化しようとして失敗した**構図を、紙とモチーフで体現。素材辞書＝`scenario/design_iconography.md`。
- **二層**: 表＝**和紙＋朱＋墨＋朱印**（可読性）／裏・固有マス＝**濃紺＋民俗×研究の融合**（勾玉＝魂・注連縄/御札＝封印・鳥居＝境界 に、**標本票/型番/バーコード/走査装置/電極**など研究ラボ要素を必ず重ねる）。**メカニクスと絵が一致**＋「研究所が伝承を標本化した」融合を全カードで維持。
- **配色**: 朱(208,86,60)・金(202,170,104)・墨(40,33,29)・和紙(239,234,223→223,216,200)・濃紺(34,44,60→15,22,34)。
- **再生成**: 別案を見たい時は `make_back_safe("dial"/"lock")` `make_back_biodoor("gate"/"mirror"/...)` `make_card_back_wa("tomoe"/"shimenawa"/"enso")` 等を `__main__` に呼び戻す。論文の配色/レイアウト探索は `design_paper_styles.py` / `design_layouts.py`。

- `00_concept.md` の「研究資料カード＝表面情報＋教授のみ読める裏面」と対応。

## デザイン仕様（再現の基盤）

### 寸法
- カード（表/裏）: 600 × 840 px
- 研究論文: 1000 × 1414 px（A4縦比 1:1.414）、余白 M=84、本文2段組（カラム幅 ≒394、段間44）

### フォント（Windows標準）
| 用途 | フォント | パス |
|------|---------|------|
| 明朝・見出し(太) | 游明朝 Demibold | `yumindb.ttf` |
| 明朝・本文 | 游明朝 | `yumin.ttf` |
| ゴシック・本文 | 游ゴシック Medium | `YuGothM.ttc` |
| ゴシック・見出し(太) | 游ゴシック Bold | `YuGothB.ttc` |
| （候補）明朝 | BIZ UD明朝 | `BIZ-UDMinchoM.ttc` |

### 配色
| 要素 | warm（証拠タグ/温） | mono（研究文書/無機質） |
|------|--------------------|------------------------|
| 地（グラデ） | クラフト紙 (234,230,220)→(206,200,187) | 白 (250,250,252)→(243,244,246) |
| インク | 濃茶 (46,41,36) | 黒に近い (28,30,34) |
| アクセント | 朱 (150,36,38) | グレー (74,80,90) |
| スタンプ | 朱・傾き-14°（機密/墨消/原本） | グレー・水平（管理） |

### 技法（PILのみ）
- 縦グラデ背景（`vgrad`）＋紙ノイズ（`paper_noise`）
- 二重罫＋四隅L字コーナー飾り（`corner`、warmカードのみ）
- 墨消し: 本文ブロックの描画y範囲を記録し、redacted時に黒矩形を上掛け（同一テキストで原本/墨消しを両立）
- スタンプ: 別レイヤーに角枠＋文字→回転→alpha合成（`stamp`）

## 経緯メモ
- ファイルI/Oの状態同期遅延が頻発した環境のため、生成は都度スクリプト実行で再現する方針。
- 正解XML構造はソース推論でなく**実機エクスポートの実物zipとの照合**で確定した（最短・最確実）。
