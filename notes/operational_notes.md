# 運用メモ（auto memory から移植）

> グローバル auto memory に保存されていた運用ルールを、プロジェクト可搬性のために複製。

## 1. 設計セッションの進め方

物語世界・人格設定・プロット等の設計決定を行う際は、複数の選択肢をテキストで列挙してユーザーに答えさせるのではなく、**AskUserQuestion を一問ずつ個別に発行**して決定していく。

- 複数の論点が絡む設計決定では、最も根幹のものから順に一つずつ確認
- 一度に複数質問をまとめない（最大1項目ずつ）
- 前の回答を受けて次の問いを派生させる、対話的な設計プロセスで進める
- 選択肢は「本命（Recommended）」を最初に置き、対抗案を2-3つ提示するのが基本形

## 2. ロールプレイのラリー形式

ロールプレイのラリー（対話プロトタイプ）では以下の形式で進行する：

- **括弧なしのテキスト** → 朔のセリフ・行動・内心（作中の描写）
- **（括弧付きテキスト）** → 演出指示・方向性の修正・メタ的なフィードバック

括弧付きの指示があった場合は直前の応答を調整するか、次の応答に反映する。
