# data/ ── 配布物本文データ（A'層）

配布物（`dist/`）に載る本文テキストの**正史**。生成スクリプト（`tools/`）はここから読み込む。

## 運用

- **1成果物群=1ファイル**
  - 例: `evidence_p1.yaml`＝第1調査の証拠カード全枚、`charsheet_pc1.yaml`＝PC1のシート全状態（初期＋更新）
- 形式は機械可読なものをプロジェクト開始時に1つ選ぶ。推奨: **YAML**
  （フォールバック: dict のみを含むデータ専用 `.py` — 依存ゼロで既存レンダラの引数形式にそのまま渡せる）
- ここに書くのは「印刷される文字列」とレンダラに渡す描画パラメータ（variant 等）のみ。
  配置理由・設計意図は `scenario/` 側（手がかり設計・証拠カタログ）に書く
- `scenario/` 側に同じ本文を複製しない。設計書からは「本文は `data/xxx` を参照」と書く
  （事実オーナー制: `scenario/90_changes.md` の表を参照）
