# data/ ── 配布物本文データ（A'層）

配布物（`dist/`）に載る本文テキストの**正史**。生成スクリプト（`tools/`）はここから読み込む。

## 運用

- **階層は `dist/` をミラーする**（2026-07-13 確定）: 生成先と同じカテゴリ配下にデータを置く
  - 例: `dist/配布物/共通/導入シート.png` ← `data/配布物/共通/intro_sheet.md`
  - ファイル名（basename）は ASCII（エンコーディング事故防止）。ディレクトリ名は dist/ の既存慣例に従う
- **1成果物群=1ファイル**
  - 例: `evidence_p1.md`＝第1調査の証拠カード全枚、`charsheet_pc1.md`＝PC1のシート全状態（初期＋更新）
- 形式: **Markdown（セクション見出し＋キー行）で確定**（2026-07-13。手編集のしやすさを優先。パース規約は各データファイル冒頭の注記とレンダラを参照）
- ここに書くのは「印刷される文字列」とレンダラに渡す描画パラメータ（variant 等）のみ。
  配置理由・設計意図は `scenario/` 側（手がかり設計・証拠カタログ）に書く
- `scenario/` 側に同じ本文を複製しない。設計書からは「本文は `data/xxx` を参照」と書く
  （事実オーナー制: `scenario/90_changes.md` の表を参照）
