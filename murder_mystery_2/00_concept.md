## 1. 着想元: Look Outside

### 基本情報

- **開発**: Francis Coulombe / **パブリッシャー**: Devolver Digital
- **リリース**: 2025年3月（Steam 評価98%「圧倒的に好評」）
- **ジャンル**: 見下ろし型探索 + EarthBound風ターン制RPG + サバイバルホラー
- **開発者が明言した影響元**: Sweet Home（1989 FC版）、EarthBoundの戦闘メカニクス
- ※「伊藤潤二 地獄星レミナが影響元」という情報はゲームメディアのレビュアーによる類推であり、開発者本人の発言ではない（CGMagazine が "probably most directly" と推測）

### ゲーム概要

太陽系を通過する宇宙的存在「The Visitor」が地球に接近。外を見た者がグロテスクな怪物に変貌する世界で、主人公サムが15日間アパートで生き延びつつ、天文学者ジャスパー率いるグループと協力してVisitorの物理的記録を集める。

### 恐怖の構造（本作への転用）

| 要素 | Look Outside | 本作での転用 |
|------|-------------|-------------|
| **安全/危険の落差** | アパート内は安全、外に出ると時間が進む | マンション内＝相対安全、外出＝死/変異リスク |
| **敵距離システム** | 遠くでぼやけ→接近で真の姿が判明 | 段階的情報開示（ラウンド進行で外の状況が悪化） |
| **罪悪感システム** | 生存のために道徳を犠牲にするか | 「誰が外に出るか」の交渉がジレンマ化 |
| **15日制限** | 新エリア探索で日数消費 | 5〜7ラウンドの日数制限 |
| **マルチエンディング** | 行動の蓄積で分岐 | 各PLの秘匿目的の達成度で結末が分岐 |
| **認識＝破滅** | Visitorを正確に見ると即ゲームオーバー | 「外を見る」と「外に出る」が別リスクとして存在 |

## 2. コズミックホラーの定義的要素

本シナリオが拠って立つジャンル特性を整理する。

| 要素 | 内容 |
|------|------|
| **人間中心主義の否定** | 宇宙において人間は取るに足らない存在。特別でも主役でもない |
| **認識の限界と破滅** | 真実を知ること自体が狂気や破滅をもたらす。知ることが救済ではなく呪い |
| **不可知性** | 恐怖の対象が人間の知覚・言語・論理では記述不可能 |
| **無関心な宇宙** | 宇宙的存在は人間に悪意すら持っていない。認識すらしていない |
| **無力感と不可逆性** | 戦って勝てない。交渉できない。せいぜい「少し遅らせる」か「見なかったことにする」 |

> 恐怖の正体は「人間が宇宙にとって無意味であるという事実そのもの」。
> Look Outsideはここにボディホラー（肉体変容）を掛け合わせ、認識の破滅が精神だけでなく肉体にも及ぶ構造。

## 3. 世界観の骨格

> ある日、空に「何か」が現れた。外を見た人間が次々と異変を起こし始め、政府が屋内退避指示を出す。マンションの住人たちは窓を塞ぎ、管理人の指示のもとで共同生活を強いられる。物資は減り、外の状況は悪化し、いつ終わるかも分からない。各人にはこの状況下でも**どうしても達成しなければならないこと**がある。

### 空間の二層構造

| 層 | 場所 | リスク |
|---|------|--------|
| **マンション内** | 自室・廊下・他の部屋・共用部 | 比較的安全だが他PLとの接触が発生 |
| **マンション外** | 建物の外 | **死 or 変異**。見るだけでも別種のリスク |

### 行動のグラデーション

```
自室に籠る（安全だが情報もリソースも得られない）
  → マンション内を動く（交渉・探索はできるがイベントリスク）
    → 外に出る（物資・情報が得られるが死/変異のリスク）
```

- **「外を見る」と「外に出る」は別のリスク**
- 一部イベントでは**外出が強制される** → 「誰が行くか」が交渉の最大焦点

## 4. ゲーム構造

### ゲームの型

**交渉＋課題解決ベース**。犯人探し（推理）ではなく、各PLが自分の秘匿目的を達成するために他PLと交渉・協力・駆け引きする。

| 要素 | 設計 |
|------|------|
| **勝利条件（全体）** | 全員が生存ラインを維持してゲーム終了を迎える |
| **勝利条件（個人）** | 各PLの秘匿目的の達成度 |
| **核の緊張** | 全体目標と個人目標が**微妙にずれている** |
| **設計上の急所** | 4人の秘匿目的間に**依存関係**が必要（独立だと各自黙々と最適行動するだけになる） |

### ラウンド構造（1ラウンド＝1日）

```
朝 ── 状況報告（リソース残量・外の変化をアプリが表示）
昼 ── 行動フェーズ（探索・交渉・外出の意思決定）
夜 ── 消費＋イベント（リソース減少・強制イベント発生）
```

5〜7ラウンドでゲーム終了。

### GM（管理人）の役割

**ハイブリッド型**:
- **アプリが担当**: リソース管理・状態表示・定型イベント発生・ラウンド進行
- **人間GMが担当**: 想定外の行動への裁定・演出・NPC（管理人）としてのロールプレイ

## 5. マダミスとTRPGの構造差異（設計上の前提知識）

覗きの代（マダミス）の経験をTRPGに転用するにあたり、両形式の根本的な違いを整理する。

| 設計軸 | マダミス | TRPG |
|--------|---------|------|
| **物語の所在** | シナリオに全て書かれている。真相は開始前に確定 | プレイ中に生成される |
| **PL行動の自由度** | 限定的（議論・投票・情報公開等） | 原則なんでもできる |
| **情報設計** | **分配型**（事前に設計し交換させる） | **探索型**（PLが能動的に調べて発見） |
| **GMの役割** | 進行役（タイマー管理・ルール裁定） | 世界そのもの（NPC即興・想定外対応・演出） |
| **時間設計** | 厳密なフェーズ制 | 流動的（PLの行動次第） |
| **再プレイ性** | ほぼゼロ | 高い（同シナリオでも展開が変わる） |
| **シナリオ作者の仕事** | 全情報の矛盾なき設計（パズル設計） | 状況と動機の種まき（壊れにくい構造設計） |

### 覗きの代から転用できるスキル

- 情報の非対称設計
- フェーズ管理
- 固有スキルの「通常ルールを1箇所破る」設計
- 証拠カードによる段階的情報開示

### 本作で新たに必要な設計軸

- PLが想定外の行動をとることを前提にした「壊れにくい構造」
- どの順番で情報が出ても論理が成立する設計
- **リソースの枯渇速度バランス**（交渉が発生するためには「足りないが詰まない」調整が必要）

## 6. ギミック設計の方向性

予想できないギミックを生む3つのアプローチ。

### 6-1. メタレイヤーの破壊

PLの立場そのものが変わる系。覗きの代のオンタイム乗り換えがこれに該当。
- 強力だが1シナリオ1回。2作目で読まれるリスク

### 6-2. 物理/インターフェースの介入

ブラウザアプリの機能を活用：
- リソースUIが条件で変化（持つ人によって見える情報が異なる）
- 通信制限（特定ラウンドで他PLの状態が見えなくなる）
- アプリ側でしか見えない情報（GM画面とPL画面の差異）

### 6-3. PLの認知バイアスの利用

最も予想されにくい方向：
- PLが「勝った」と思った後にひっくり返す
- 情報を集めるほど状況が悪化するメカニクス（Look Outside的「見たら変異」構造）
- リソースを溜め込む行為自体がリスクを招く

## Appendix A: 旧構想（VCチャンネル分離型）

> 以下は初期構想時の検討内容。現行方針（リソース管理型）への転換に伴い、参考資料として保存。

### コアコンセプト

PLが別々の通話チャンネル（Discord VC等）に分かれて探索する。
物理的な孤立を通信環境で強制し、ホラーの恐怖を増幅する。

### この形式でしか出せない恐怖

1. **「他のPLに何が起きているか分からない」** — 隣の部屋で悲鳴が聞こえた気がするが確認する手段がない
2. **「合流時の違和感」** — 「あいつ、さっきと言ってることが違う」「本当にさっきの部屋にいたのか？」という疑念が自然発生
3. **「通信そのものが信用できない」** — 全体チャンネルで呼びかけても返事がない。返事があっても本人か分からない

### GM負荷問題

4人が別チャンネル → GMは4つの並行セッションを同時進行。人間1人では物理的に無理。
現行方針ではリソース管理アプリが分離中の進行を代替し、この問題を構造的に回避する。
